### 文件命名规则
本次迁移沿用统一的特殊符号转写规则：

1. `?` -> `-p`
2. `!` -> `-bang`
3. `->` -> `-to-`
4. `=` -> `-eq`
5. `*` -> `-star`
6. `/` -> `-slash-`
7. 关系运算符中的 `<` / `>` / `<=` / `>=` 按语义写法转写
8. 单独的算术运算符名称采用语义化命名

本次具体文件名示例：
- `hash-table-contains?` -> `hash-table-contains-p-test.scm`
- `hash-table-empty?` -> `hash-table-empty-p-test.scm`
- `hash-table=?` -> `hash-table-eq-p-test.scm`
- `hash-table-ref/default` -> `hash-table-ref-slash-default-test.scm`
- `hash-table-update!/default` -> `hash-table-update-bang-slash-default-test.scm`
- `hash-table-set!` -> `hash-table-set-bang-test.scm`
- `hash-table->alist` -> `hash-table-to-alist-test.scm`
- `alist->hash-table` -> `alist-to-hash-table-test.scm`

### Why
1. `tests/goldfish/liii/hash-table-test.scm` 聚合了大量独立的哈希表 API 测试，不利于维护和定位问题。
2. 拆分为单函数测试文件后，便于单独执行、回归和审查。
3. 与最近的 `string` / `vector` / `base` / `ascii` / `either` 迁移方式保持一致，目录结构更统一。
4. 对原文件里没有独立注释块的函数，补齐了统一格式的文档注释，便于后续维护。

### How
1. 从原 `hash-table-test.scm` 中按函数边界提取测试代码。
2. 按“一个函数一个文件”的方式迁移到 `tests/liii/hash-table/`。
3. 对缺少说明的函数先结合 `goldfish/srfi/srfi-125.scm` 的实现语义理解其职责，再补齐中文文档块。
4. 保留原有测试覆盖范围，尽量只做结构迁移，不随意改测试语义。
5. 删除旧的聚合测试文件 `tests/goldfish/liii/hash-table-test.scm`。
6. 新增 `tests/liii/hash-table-test.scm`，仅用于说明测试已迁移到目录中。
7. 使用项目真实测试入口 `bin/goldfish -m r7rs` 逐个执行 `tests/liii/hash-table/` 下的所有测试文件，确认迁移后全部通过。
