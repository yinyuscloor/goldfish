## 2026-03-30 拆分 json-test.scm

### What
将 `tests/goldfish/liii/json-test.scm` 中聚合的 `(liii json)` 测试，按函数拆分到 `tests/liii/json/` 目录中，并新增轻量入口文件 `tests/liii/json-test.scm`。

本次迁移覆盖的函数包括：
1. `json-null?`、`json-object?`、`json-array?`、`json-string?`
2. `json-number?`、`json-integer?`、`json-float?`、`json-boolean?`
3. `json-ref`、`json-get-or-else`、`json-keys`、`json-contains-key?`
4. `string->json`、`json-string-escape`、`json->string`
5. `json-set`、`json-ref-boolean`、`json-reduce`、`json-push`、`json-drop`

### 注释格式
每个测试文件都使用统一的文档块注释格式：函数名、简要说明、`语法 / 参数 / 返回值 / 注意 / 示例 / 错误处理`。

### 文件命名规则
本次迁移沿用统一的特殊符号转写规则：
1. `?` -> `-p`
2. `!` -> `-bang`
3. `->` -> `-to-`
4. `=` -> `-eq`
5. `*` -> `-star`
6. `/` -> `-slash-`
7. 关系运算符中的 `<` / `>` / `<=` / `>=` 按语义写法转写

### Why
1. 原 `tests/goldfish/liii/json-test.scm` 同时包含类型谓词、解析、编码和变换 API 的测试，单文件维护成本较高。
2. 拆分后可以按函数粒度单独执行、回归和审查。
3. 迁移到 `tests/liii/json/` 后，与现有 `string` / `vector` / `ascii` / `base` / `either` / `enum` 的目录结构保持一致。
4. 每个文件只保留对应函数的断言，减少不同 API 测试相互混杂。

### How
1. 先按函数边界整理原有测试，将混在 `json-ref` 附近的 `string->json` 断言迁回 `string->json-test.scm`。
2. 对每个目标函数创建独立测试文件，并补齐统一的文档注释。
3. 使用轻量入口文件 `tests/liii/json-test.scm` 指向新的目录位置。
4. 删除旧聚合文件 `tests/goldfish/liii/json-test.scm`。
5. 使用项目真实测试入口 `bin/goldfish -m r7rs` 逐个执行 `tests/liii/json/` 下的测试文件做回归验证。
