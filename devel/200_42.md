## 2026-03-26 新增启动时库搜索路径扩展机制

### What
新增两个全局启动参数：

1. `-I DIR`：将 `DIR` 前置到库搜索路径
2. `-A DIR`：将 `DIR` 追加到库搜索路径

同时，启动时会自动扫描 `$HOME/.local/goldfish/` 下的一级子目录：

- 目录名必须匹配 `xxx-yyy`
- 目录内递归存在至少一个 `.scm` 文件
- 命中后自动前置到 `*load-path*`

### Why
当前 Goldfish 的 `(import ...)` 已经基于 `*load-path*` 查找库文件，但缺少一套正式的启动时扩展入口。  
新增 `-I` / `-A` 后，可以在不修改内置标准库目录的情况下，按需挂载本地库根目录或外部工具目录。

自动扫描 `$HOME/.local/goldfish/` 后，用户可以把本地扩展库放到统一位置，启动时自动生效。

### How
本次实现只修改启动层，不修改 `boot.scm` 的 `define-library` / `import` 宏逻辑。

主要实现点：

1. 在 `src/goldfish.hpp` 中新增统一的启动参数解析结构 `StartupCliOptions`
2. 将全局参数解析扩展为同时支持 `--mode`、`-I`、`-A`
3. 新增 `load-path` 辅助函数：
   - 读取当前 `*load-path*`
   - prepend 新目录
   - append 新目录
   - 去重并保持顺序稳定
4. 在加载 `boot.scm` 和默认 mode 导入前，先应用 `-I` / 自动扫描 / `-A`
5. 更新 CLI 帮助和 README 文档

### 目录约定
`-I` / `-A` 传入的是“库根目录”，而不是命名空间目录本身。

例如，如果目录结构为：

```text
~/.local/goldfish/liii-goldfix/
  liii/
    goldfix.scm
```

那么导入：

```scheme
(import (liii goldfix))
```

时，正确写法是：

```bash
bin/gf -I ~/.local/goldfish/liii-goldfix
```


### 兼容性说明
- 继续保留 `--mode` / `-m` 放在命令后面的旧行为
- 手工 `-I` / `-A` 的路径要求由 shell 先完成 `~` 展开，因此推荐写法为：

```bash
bin/gf -I ~/.local/goldfish/liii-goldfix
```
