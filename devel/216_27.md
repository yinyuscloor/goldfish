### 注释格式
每个测试文件都使用统一的文档块注释格式：函数名、简要说明、`语法 / 参数 / 返回值 / 注意 / 示例 / 错误处理`。

### 文件命名规则
本次迁移沿用统一的特殊符号转写规则：
1. `?` -> `-p`
2. `!` -> `-bang`
3. `->` -> `-to-`
4. `=` -> `-eq`
5. `<` / `>` / `<=` / `>=` 按语义写法转写为 `lt` / `gt` / `le` / `ge`

### Why
1. 原 `tests/goldfish/liii/time-test.scm` 聚合了 30+ 个函数的测试，文件很大（1963 行），定位和维护成本较高。
2. 拆成单文件后，可以像 `list`、`vector`、`json`、`range`、`sort`、`path`、`os` 目录一样单独运行和回归。
3. 入口说明文件保留模块级导航，同时避免旧聚合文件继续承担测试职责。
4. 统一每个测试文件的文档格式，便于后续继续迁移同类模块。

### How
1. 参考已有 `tests/liii/sort/` 和 `tests/liii/path/` 的拆分方式，按函数边界切分测试。
2. 对测试文件使用统一的注释结构，使用 `;;` 行注释。
3. 文件命名按统一规则转换特殊符号：
   - `?` -> `-p`
   - `!` -> `-bang`
   - 关系运算符按语义转写
4. 保留原有测试覆盖范围，只调整目录结构和说明文字，不改测试语义。
5. 迁移完成后，逐个运行新测试文件，总测试用例数：约 200+ 个。
