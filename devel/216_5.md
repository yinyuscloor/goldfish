### 文件命名规则
本次迁移沿用统一的特殊符号转写规则：

1. `?` -> `-p`
   - `ascii-codepoint?` -> `ascii-codepoint-p-test.scm`

2. `!` -> `-bang`

3. `->` -> `-to-`
   - `ascii-control->graphic` -> `ascii-control-to-graphic-test.scm`

4. `=` -> `-eq`
   - `ascii-ci=?` -> `ascii-ci-eq-p-test.scm`

5. `*` -> `-star`

6. `/` -> `-slash-`
   - 若未来函数名中出现 `/`，统一按该规则处理，以保持可读性和可逆性

7. 关系运算符中的 `<` / `>` / `<=` / `>=` 按语义写法转写
   - `<` -> `lt`
   - `>` -> `gt`
   - `<=` -> `le`
   - `>=` -> `ge`

### Why
1. `tests/goldfish/liii/ascii-test.scm` 聚合了过多独立函数测试，不利于维护和定位问题。
2. 拆分为单函数测试文件后，便于单独执行、回归和审查。
3. 与最近的 `string` / `vector` / `list` 迁移方式保持一致，目录结构更统一。
4. 每个测试文件都补齐了统一格式的文档注释，便于后续维护。

### How
1. 从原 `ascii-test.scm` 中按函数边界提取测试代码。
2. 按“一个函数一个文件”的方式迁移到 `tests/liii/ascii/`。
3. 为每个新文件补齐统一的文档块注释。
4. 删除旧的聚合测试文件 `tests/goldfish/liii/ascii-test.scm`。
5. 新增 `tests/liii/ascii-test.scm`，仅用于说明测试已迁移到目录中。
6. 使用项目真实测试入口 `bin/goldfish -m r7rs` 逐个执行 `tests/liii/ascii/` 下的所有测试文件，确认迁移后全部通过。
