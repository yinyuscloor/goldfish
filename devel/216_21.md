### 注释格式
每个测试文件都使用统一的文档块注释格式：函数名、简要说明、`语法 / 参数 / 返回值 / 注意 / 示例 / 错误处理`。

### 文件命名规则
本次迁移沿用统一的特殊符号转写规则：
1. `?` → `-p`
2. `!` → `-bang`
3. `->` → `-to-`
4. `=` → `-eq`
5. `*` → `-star`
6. `/` → `-slash-`
7. 关系运算符中的 `<` / `>` / `<=` / `>=` 按语义转写为 `lt` / `gt` / `le` / `ge`

### Why
1. 原 `tests/goldfish/liii/path-test.scm` 同时包含构造、谓词、转换、路径操作和文件操作的测试，单文件维护成本较高（755 行）。
2. 拆分后可以按函数粒度单独执行、回归和审查。
3. 迁移到 `tests/liii/path/` 后，与现有 `string` / `vector` / `ascii` / `base` / `either` / `enum` / `iset` / `json` / `njson` 等目录结构保持一致。
4. 每个文件只保留对应函数的断言，减少不同 API 测试相互混杂。

### How
1. 先按函数边界整理原有测试，将混在 `path-join` 附近的 `path-read-text` 断言迁回 `path-read-text-test.scm`。
2. 对每个目标函数创建独立测试文件，并补齐统一的文档注释。
3. 使用轻量入口文件 `tests/liii/path-test.scm` 指向新的目录位置。
4. 删除旧聚合文件 `tests/goldfish/liii/path-test.scm`。
5. 使用 `bin/gf` 逐个执行 `tests/liii/path/` 下的测试文件做回归验证。
