### 拆分规范
参考 `string-replace-test.scm` 的格式：
1. **无 Apache License 头**：测试文件直接从 `(import ...)` 开始
2. **简洁的 import**：`(import (liii check) (liii string))`
3. **文档注释**：使用 `;;` 开头的行注释，包含函数说明、参数、返回值、错误处理等
4. **测试用例**：包括基本功能、边界条件、Unicode支持、错误处理等
5. **结尾**：`(check-report)`

### 命名规则
测试文件命名为 `{函数名}-test.scm`，例如：
- `string-join-test.scm`
- `string-null-p-test.scm`（Windows 文件名不支持 `?`，用 `-p` 代替）
- `string-take-test.scm`（string-take 和 string-take-right 可以放在一起）

### Why
1. 单个测试文件过大（32510+ tokens），不利于维护
2. 拆分后每个函数有独立的测试文件，便于定位和修复问题
3. 参考 `string-replace-test.scm` 的格式，保持测试风格一致性
4. 便于后续回归测试和代码审查

### How
1. 从 `string-test.scm` 中提取每个函数的文档块和测试代码
2. 按照 `string-replace-test.scm` 的格式重新组织
3. 移除 Apache License 头（根据 CLAUDE.md 规范）
4. 保持原有的测试用例和覆盖范围

## 2026-03-31 改进 string-split-test.scm 代码格式

### What
改进 `tests/liii/string/string-split-test.scm` 的代码格式，统一 `check` 语句的列表结果格式。

### Why
原文件中的 `check` 语句存在格式不一致问题：
```scheme
; 原格式（换行拆分裂表）
(check (string-split "a,b,c" ",") => '(
"a" "b" "c"))

; 改进后（单行格式）
(check (string-split "a,b,c" ",") => '("a" "b" "c"))
```

改进后的格式更紧凑、可读性更好，且符合 Scheme 代码的惯用写法。

### How
统一所有 `check` 语句的期望结果格式为单行列表形式，保持 2 空格缩进。
