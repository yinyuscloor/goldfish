## 2026-03-31 修复 Windows 平台路径处理问题

### What
修复 `gf test` 命令在 Windows 平台上的路径处理问题：

1. **修复路径分隔符识别**：在 `parse-test-args` 函数中，除了识别 `/` 之外，在 Windows 平台上还识别 `\` 作为路径分隔符
2. **修复目录匹配**：在 `filter-test-files` 函数中，Windows 平台上将用户输入的正斜杠转换为反斜杠，以匹配实际的文件路径
3. **添加测试用例**：在 `path-dir-p-test.scm` 中添加使用正斜杠的相对路径测试，验证跨平台路径处理

### Why
- Windows 平台使用反斜杠 `\` 作为路径分隔符，与 Unix 系统的正斜杠 `/` 不同
- 用户可能输入正斜杠路径（如 `tests/liii/path`），需要正确匹配实际的文件路径
- 确保 `gf test` 命令在 Windows 上能正确处理各种路径格式

### How
修改 `tools/goldtest/liii/goldtest.scm`：
1. 在 `parse-test-args` 函数中，Windows 平台额外检查 `\` 分隔符
2. 在 `filter-test-files` 函数的 `dir` 处理分支中，Windows 平台将 `/` 替换为 `\` 后再匹配

修改 `tests/liii/path/path-dir-p-test.scm`：
- 添加使用正斜杠的相对路径测试用例

## 2026-03-30 简化 gf test 命令参数

### What
简化 `gf test` 命令的参数解析逻辑，移除 `--only` 选项，改为根据参数类型自动判断匹配方式：

1. **包含 `/` 的路径** - 按路径处理：
   - 如果是存在的文件，直接运行该文件
   - 如果是存在的目录，运行该目录下所有 `*-test.scm` 文件
   - 如果不存在，按模糊匹配处理

2. **以 `.scm` 结尾但不是路径** - 按文件名精确匹配
   - 例：`json-test.scm` 匹配所有名为 `json-test.scm` 的文件

3. **其他字符串** - 按路径模糊匹配
   - 例：`json` 匹配路径中包含 `json` 的所有测试文件

### Why
- `--only` 选项的语法比较冗长
- 根据参数形式自动判断更符合直觉
- 与 `gf run` 命令的参数处理方式保持一致

### How
修改 `tools/goldtest/liii/goldtest.scm`：
1. 新增 `parse-test-args` 函数，根据参数特征返回 `(type . value)`：
   - `'file` - 直接运行该文件
   - `'dir` - 运行目录下所有测试
   - `'filename` - 按文件名匹配
   - `'pattern` - 按路径模糊匹配
2. 修改 `filter-test-files` 函数，根据类型进行过滤
3. 新增 `display-filter-info` 函数，显示当前匹配模式
4. 修改 `run-goldtest` 函数，使用新的解析逻辑

修改 `src/goldfish.hpp`：
- 更新帮助文本，移除 `--only` 选项，说明新的参数用法

新增 `tests/liii/goldtest/parse-test-args-test.scm`：
- 单元测试验证参数解析逻辑
- 测试文件通过 `import (liii goldtest)` 使用库函数

修改 `tools/goldtest/liii/goldtest.scm`：
- 改为 `define-library (liii goldtest)` 格式
- 导出 `parse-test-args`, `filter-test-files`, `find-test-files`, `run-goldtest`, `main`
- 增加 `main` 函数作为程序入口点
