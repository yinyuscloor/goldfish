## 2026-03-30 拆分 os-test.scm

### What
将 `tests/goldfish/liii/os-test.scm` 按"一个函数一个测试文件"的方式迁移到 `tests/liii/os/` 目录，并新增轻量入口说明文件 `tests/liii/os-test.scm`。

本次拆分覆盖的函数如下：

1. `os-type` -> `os-type-test.scm`
2. `os-linux?` -> `os-linux-p-test.scm`
3. `os-macos?` -> `os-macos-p-test.scm`
4. `os-windows?` -> `os-windows-p-test.scm`
5. `os-call` -> `os-call-test.scm`
6. `access` -> `access-test.scm`
7. `putenv` -> `putenv-test.scm`
8. `getenv` -> `getenv-test.scm`
9. `unsetenv` -> `unsetenv-test.scm`
10. `os-temp-dir` -> `os-temp-dir-test.scm`
11. `mkdir` -> `mkdir-test.scm`
12. `rmdir` -> `rmdir-test.scm`
13. `remove` -> `remove-test.scm`
14. `rename` -> `rename-test.scm`
15. `listdir` -> `listdir-test.scm`
16. `getcwd` -> `getcwd-test.scm`

### Why
1. 旧的 `os-test.scm` 聚合了多个函数的测试，文件较大，定位和维护成本较高。
2. 拆成单文件后，可以像 `bitwise`、`list`、`vector` 目录一样单独运行和回归。
3. 入口说明文件保留模块级导航，同时避免旧聚合文件继续承担测试职责。
4. 统一每个测试文件的文档格式，便于后续继续迁移同类模块。

### How
1. 参考 `tests/liii/bitwise/` 的拆分方式，按函数边界切分测试。
2. 对测试文件使用统一的注释结构。
3. 文件命名按统一规则转换特殊符号：`?` -> `-p`。
4. 保留原有测试覆盖范围，只调整目录结构和说明文字，不改测试语义。
5. 删除旧的 `tests/goldfish/liii/os-test.scm` 聚合文件（将在后续提交中处理）。
