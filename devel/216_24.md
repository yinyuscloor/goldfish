## 2026-03-30 拆分 sort-test.scm

### What
将 `tests/goldfish/liii/sort-test.scm` 按"一个函数一个测试文件"的方式迁移到 `tests/liii/sort/` 目录，并新增轻量入口说明文件 `tests/liii/sort-test.scm`。

本次拆分覆盖的函数如下（共 11 个文件）：

**List operations（6 个）**
1. `list-sorted?` -> `list-sorted-p-test.scm` (6 个测试用例)
2. `list-sort` -> `list-sort-test.scm` (8 个测试用例)
3. `list-sort!` -> `list-sort-bang-test.scm` (9 个测试用例)
4. `list-stable-sort` -> `list-stable-sort-test.scm` (7 个测试用例)
5. `list-stable-sort!` -> `list-stable-sort-bang-test.scm` (7 个测试用例)
6. `list-merge` -> `list-merge-test.scm` (8 个测试用例)
7. `list-merge!` -> `list-merge-bang-test.scm` (7 个测试用例)

**Vector operations（4 个）**
8. `vector-sorted?` -> `vector-sorted-p-test.scm` (10 个测试用例)
9. `vector-sort` -> `vector-sort-test.scm` (8 个测试用例)
10. `vector-stable-sort` -> `vector-stable-sort-test.scm` (7 个测试用例)
11. `vector-merge` -> `vector-merge-test.scm` (8 个测试用例)

### 注释格式
每个测试文件都使用统一的文档块注释格式：函数名、简要说明、`语法 / 参数 / 返回值 / 注意 / 示例 / 错误处理`。

### 文件命名规则
本次迁移沿用统一的特殊符号转写规则：
1. `?` -> `-p`
2. `!` -> `-bang`
3. `->` -> `-to-`
4. `=` -> `-eq`
5. `*` -> `-star`
6. `/` -> `-slash-`
7. 关系运算符中的 `<` / `>` / `<=` / `>=` 按语义写法转写

### Why
1. 原 `tests/goldfish/liii/sort-test.scm` 聚合了 11 个函数的测试，文件较大（131 行），定位和维护成本较高。
2. 拆成单文件后，可以像 `list`、`vector`、`json`、`range`、`os` 目录一样单独运行和回归。
3. 入口说明文件保留模块级导航，同时避免旧聚合文件继续承担测试职责。
4. 统一每个测试文件的文档格式，便于后续继续迁移同类模块。

### How
1. 参考已有 `tests/liii/range/` 和 `tests/liii/os/` 的拆分方式，按函数边界切分测试。
2. 对测试文件使用统一的注释结构，使用 `;;` 行注释。
3. 文件命名按统一规则转换特殊符号：
   - `?` -> `-p`
   - `!` -> `-bang`
4. 保留原有测试覆盖范围，只调整目录结构和说明文字，不改测试语义。
5. 迁移完成后，逐个运行新测试文件，总测试用例数：约 95 个。
