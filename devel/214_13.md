## 2026/3/6 njson 新增 4 个 Scheme 容器转换 API

### What
1. Scheme API 扩展：
   - 在 `goldfish/liii/njson.scm` 新增并导出 4 个 typed conversion API：
     - `njson-object->alist`
     - `njson-object->hash-table`
     - `njson-array->list`
     - `njson-array->vector`
   - Scheme 层仅保留 root 类型校验与 primitive 转发。
2. C++ 运行时扩展：
   - 在 `src/goldfish.hpp` 新增递归转换链路，直接从 `nlohmann::json` 构造 Scheme 容器；
   - 新增并注册 `g_njson-object->alist` / `g_njson-object->hash-table` /
     `g_njson-array->list` / `g_njson-array->vector`；
   - 抽出 `njson_scalar_value_to_scheme` 统一处理标量到 Scheme 的映射。
3. 测试扩展与整理：
   - 在 `tests/goldfish/liii/njson-test.scm` 引入 `(liii hash-table)`；
   - 为 4 个 API 分别新增独立测试块与文档说明；
   - 覆盖递归转换、空容器、类型错误、已释放句柄、结果脱离句柄生命周期后的可用性。

### Why
1. `njson` 缺少直接转为原生 Scheme 容器的 API，调用方只能先走 `njson->json`，再自己继续拆结构，使用成本偏高。

### How
1. 新增两组递归转换模式：
   - `alist_list`：object -> alist，array -> list
   - `hash_vector`：object -> hash-table，array -> vector
2. 新增统一入口 `njson_run_structure_conversion`：
   - 执行 owner-thread 检查；
   - 校验 handle 合法性与释放状态；
   - 校验 root 必须是 object 或 array；
   - 按目标模式递归构造 Scheme 结构。
3. 标量映射规则统一为：
   - `null -> 'null`
   - `boolean -> #t/#f`
   - integer/real/string 保持自然映射
4. `alist/list` 家族中的空 object 特判为 `'(())`，避免与空 array `()` 混淆。
5. Scheme 层 wrapper 负责把错误尽早收敛为：
   - `njson-object->alist: json must be njson object-handle`
   - `njson-object->hash-table: json must be njson object-handle`
   - `njson-array->list: json must be njson array-handle`
   - `njson-array->vector: json must be njson array-handle`
6. 测试层按 API 拆分，不再把 4 个转换接口混放在同一大段里，便于后续继续扩展单独行为说明。

### 兼容性说明
1. 新增 API：
   - `njson-object->alist`
   - `njson-object->hash-table`
   - `njson-array->list`
   - `njson-array->vector`
2. 这 4 个接口都是 typed conversion：
   - object family 只接受 njson object-handle
   - array family 只接受 njson array-handle
3. 多层 JSON 会递归转换：
   - `njson-object->alist` / `njson-array->list` 返回 alist/list 家族
   - `njson-object->hash-table` / `njson-array->vector` 返回 hash-table/vector 家族
4. `njson-object->alist` / `njson-array->list` 中的空 object 使用 `'(())` 作为 canonical 表示；空 array 仍为 `()`。
5. 转换结果是纯 Scheme 值，不再依赖 njson 句柄生命周期。
6. 本次实现将这 4 个 API 下沉到 C++ 层，去掉了原先可能存在的 `(liii json)` 字符串桥接成本，但不改变 `njson->json` 现有语义。
