### 文件命名规则
本次迁移中，针对 Scheme 函数名里的特殊符号采用如下转写规则：

1. `?` -> `-p`
   - `vector-contains?` -> `vector-contains-p-test.scm`
   - `int-vector?` -> `int-vector-p-test.scm`

2. `!` -> `-bang`
   - `vector-set!` -> `vector-set-bang-test.scm`
   - `vector-fill!` -> `vector-fill-bang-test.scm`

3. `->` -> `-to-`
   - `list->vector` -> `list-to-vector-test.scm`
   - `vector->string` -> `vector-to-string-test.scm`

4. `=` -> `-eq`
   - `vector=` -> `vector-eq-test.scm`

5. 普通连字符 `-` 保持不变

### Why
1. `tests/goldfish/liii/vector-test.scm` 文件过大，不利于维护和定位问题。
2. 每个函数拆成独立测试文件后，更方便单独执行、回归和审查。
3. 与最近的 `string` / `list` 测试迁移方式保持一致，目录结构更统一。
4. 为每个测试文件补齐统一格式的中文文档注释，便于后续继续维护。

### How
1. 从原 `vector-test.scm` 中按函数边界提取测试代码。
2. 按“一个函数一个文件”的方式迁移到 `tests/liii/vector/`。
3. 为每个新文件补齐统一的文档块注释。
4. 删除旧的聚合测试文件 `tests/goldfish/liii/vector-test.scm`。
5. 新增 `tests/liii/vector-test.scm`，仅用于说明测试已迁移到目录中。
6. 逐个执行 `tests/liii/vector/` 下的所有测试文件，确认迁移后全部通过。
