## 2026-03-30 拆分 inexact-test.scm

### What
将 `tests/goldfish/scheme/inexact-test.scm` 中聚合在一个文件里的测试，按函数拆分到 `tests/scheme/inexact/` 目录中，并新增轻量入口文件 `tests/scheme/inexact-test.scm`。

**注意**：与之前的拆分不同，本次拆分的目标目录是 `tests/scheme/inexact/` 而非 `tests/liii/`，因为 `(scheme inexact)` 是 R7RS 标准库而非 liii 扩展库。

本次迁移覆盖的函数包括：

1. `nan?` -> `nan-p-test.scm`
2. `sqrt` -> `sqrt-test.scm`
3. `s7-sqrt` -> `s7-sqrt-test.scm`
4. `infinite?` -> `infinite-p-test.scm`
5. `finite?` -> `finite-p-test.scm`
6. `exp` -> `exp-test.scm`
7. `log` -> `log-test.scm`
8. `sin` -> `sin-test.scm`
9. `cos` -> `cos-test.scm`
10. `tan` -> `tan-test.scm`
11. `asin` -> `asin-test.scm`
12. `acos` -> `acos-test.scm`
13. `atan` -> `atan-test.scm`

### 注释格式
每个拆分后的测试文件都采用统一的文档注释格式：

```scheme
;; 函数名
;; 函数的中文描述。
;;
;; 语法
;; ----
;; (函数名 ...)
;;
;; 参数
;; ----
;; 参数名 : 类型
;; 参数说明。
;;
;; 返回值
;; ------
;; 返回类型
;; 返回值说明。
;;
;; 注意
;; ----
;; 使用说明。
;;
;; 示例
;; ----
;; (函数名 示例参数) => 示例结果
;;
;; 错误处理
;; ------
;; 异常类型与触发条件
```

### 文件命名规则
本次迁移沿用统一的特殊符号转写规则：

1. `?` -> `-p`
2. `!` -> `-bang`
3. `->` -> `-to-`
4. `=` -> `-eq`
5. `*` -> `-star`
6. `/` -> `-slash-`
7. 关系运算符中的 `<` / `>` / `<=` / `>=` 按语义写法转写

### Why
1. `tests/goldfish/scheme/inexact-test.scm` 聚合了大量独立的数学函数测试，不利于维护和定位问题。
2. 拆分为单函数测试文件后，便于单独执行、回归和审查。
3. 保持与 liii 模块相同的拆分风格，但放在 `tests/scheme/` 目录下，区分标准库和扩展库。
4. 每个测试文件只断言对应函数的行为，避免不同 API 的断言互相混杂。

### How
1. 从原 `inexact-test.scm` 中按函数边界提取测试代码。
2. 按"一个函数一个文件"的方式迁移到 `tests/scheme/inexact/`。
3. 对缺少说明的函数补齐统一格式的文档注释。
4. 保留原有测试覆盖范围，尽量只做结构迁移，不随意改测试语义。
5. 删除旧的聚合测试文件 `tests/goldfish/scheme/inexact-test.scm`。
6. 新增 `tests/scheme/inexact-test.scm` 作为迁移后的入口说明文件。
7. 使用项目真实测试入口 `bin/goldfish -m r7rs` 逐个执行 `tests/scheme/inexact/` 下的所有测试文件，确认迁移后的测试可以独立通过。
