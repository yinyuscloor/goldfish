### 文件命名规则
本次迁移沿用统一的特殊符号转写规则：
1. `?` -> `-p`
2. `!` -> `-bang`
3. `->` -> `-to-`
4. `=` -> `-eq`
5. `*` -> `-star`
6. `/` -> `-slash-`
7. 关系运算符中的 `<` / `>` / `<=` / `>=` 按语义写法转写

### Why
1. 原 `tests/goldfish/liii/range-test.scm` 聚合了 37 个函数的测试，文件较大（924 行），定位和维护成本较高。
2. 拆成单文件后，可以像 `list`、`vector`、`alist`、`bitwise`、`json`、`iset` 目录一样单独运行和回归。
3. 入口说明文件保留模块级导航，同时避免旧聚合文件继续承担测试职责。
4. 统一每个测试文件的文档格式，便于后续继续迁移同类模块。

### How
1. 参考已有 `tests/liii/json/` 和 `tests/liii/iset/` 的拆分方式，按函数边界切分测试。
2. 对测试文件使用统一的注释结构，使用 `;;` 行注释。
3. 文件命名按统一规则转换特殊符号：
   - `?` -> `-p`
   - `->` -> `-to-`
   - 比较操作符：`<` -> `-lt-`, `<=` -> `-le-`, `>` -> `-gt-`, `>=` -> `-ge-`, `=` -> `-eq-`
4. 保留原有测试覆盖范围，只调整目录结构和说明文字，不改测试语义。
5. 迁移完成后，逐个运行新测试文件，总测试用例数：约 150+ 个。
