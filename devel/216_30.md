### 文件命名规则
本次迁移沿用统一的特殊符号转写规则：

1. `?` -> `-p`
2. `!` -> `-bang`
3. `->` -> `-to-`
4. `=` -> `-eq`
5. `*` -> `-star`
6. `/` -> `-slash-`
7. 关系运算符中的 `<` / `>` / `<=` / `>=` 按语义写法转写

### Why
1. `tests/goldfish/liii/uuid-test.scm` 虽然测试代码较少，但为了与 `string` / `vector` / `base` / `ascii` / `hash-table` / `flexvector` / `enum` / `set` 的迁移方式保持一致，目录结构更统一。
2. 拆分为单函数测试文件后，便于单独执行、回归和审查。
3. 补齐了统一的文档注释，便于后续维护。

7. 使用项目真实测试入口 `bin/goldfish -m r7rs` 执行 `tests/liii/uuid/uuid4-test.scm`，确认迁移后测试通过。
